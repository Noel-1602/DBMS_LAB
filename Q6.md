# Implementation of set operators nested queries, and join queries.
---

## Set Operators

### 1. Union 

```sql
select rlno,name from student 
where dept='Computer Science' union 
select rlno,name from student 
where dept='Data Science';
```
Note :-
1. The number of columns in both queries must be same.
2. The data type of the columns must be same. 
3. Removes duplicate rows.
---

### 2. Union All

```sql
select dept from student union all 
select dept from instructor;
```
Notes :-
1. The number of columns in both queries must be same.
2. The data type of the columns must be same. 
3. Does not remove duplicate rows.
---

### 3. Intersect

```sql
select dept from student intersect 
select dept from instructor;
```
Notes :- 
1. The number of columns in both queries must be same.
2. The data type of the columns must be same. 
3. Removes duplicate rows.
4. Similar to inner join
---

### 4. Minus / Except

```sql
select dept from student 
except select dept from instructor;
```
Notes :-
1. The number of columns in both queries must be same.
2. The data type of the columns must be same. 
3. Removes duplicate rows.
4. It selects the rows from the first query that are not present in the second query.
---

## Nested Queries

### 1. Simple Subquery

```sql
 select name from student 
 where rlno in(select rlno from enrollment);
 /* It prints names of all students who are enrolled in at least one course */
 ```
Note :-
1. The subquery is executed first and the result is used by the outer query.
2. The subquery can be used in the WHERE, FROM, and SELECT clauses.

---
### 2. Subquery with comparision operators

```sql
select name,DateOfBirth from student 
where DateOfBirth<(select avg(DateOfBirth) from student); 
/* It prints names and DateOfBirth of all students who are born before the average DateOfBirth of all students */
```
---

### 3. Correlated Subquery

```sql
select title from course c 
where (select count(*) from enrollment e 
where e.crs_id=c.crs_id)>1;
/* It prints names of all courses that have more than 1 student enrolled */
```
---

### 4. Subquery with Aggreagate Functions

```sql
select title,credits from course 
where credits = (select max(credits) from course);
/* It prints names and credits of all courses that have the maximum credits */
```
---

## Join Queries

### 1. Inner Join

```sql
select s.name,c.title 
from student s 
join enrollment e on s.rlno=e.rlno 
join course c on e.crs_id=c.crs_id;
/* It prints names and titles of all students who are enrolled in at least one course */
```
Notes :-
1. The inner join returns only the rows that have matching values in both tables.

---

### 2. Left Join

```sql
select s.name , c.title 
from student s 
left join enrollment e on s.rlno=e.rlno 
left join course c on e.crs_id=c.crs_id;
/* It prints names and titles of all students who are enrolled in at least one course */
```
Notes :-
1. The left join returns all rows from the left table, and the matching rows from the right table.
2. If there is no match in the right table, it returns NULL.
---

### 3. Right Join

```sql
select s.name , c.title 
from student s 
right join enrollment e on s.rlno=e.rlno 
right join course c on e.crs_id=c.crs_id;
/* It prints names and titles of all students who are enrolled in at least one course */
```
Notes :-
1. The right join returns all rows from the right table, and the matching rows from the left table.
2. If there is no match in the left table, it returns NULL.
---

### 4. Self Join

```sql
select a.name as instructor1,b.name as instructor2,
from instructor a, instructor b 
where a.dept=b.dept and a.inst_id<b.inst_id;
/* It prints names of all instructors who are in the same department */
```
Notes :-
1. The self join is used to join a table with itself.
2. It is used to compare two rows from the same table.
3. The condition in where clause should be such that the resulting rows are unique.
